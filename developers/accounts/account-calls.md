# Account Calls

This page shows often use Account Calls


### Calls 

|   call       |  parameters            |  description  |
|-------------------|----------|---------------|
| list_my_accounts   |   | Lists all accounts controlled by this wallet. This returns a list of the full account objects for all accounts whose private keys we possess. |
|  list_accounts |  - lowerbound: the name of the first account to return. If the named account does not exist, the list will start at the account that comes after lowerbound. <br/> - limit: the maximum number of accounts to return (max: 1000)  |  |
| list_account_balances |id: the name or id of the account whose balances you want  |  |
| register_account | -  name: the name of the account, must be unique on the blockchain. Shorter names are more expensive to register; the rules are still in flux, but in general names of more than 8 characters with at least one digit will be cheap. <br/> - owner: the owner key for the new account  <br/>- active: the active key for the new account <br/> - registrar_account: the account which will pay the fee to register the user <br/> - referrer_account: the account who is acting as a referrer, and may receive a portion of the user’s transaction fees. This can be the same as the registrar_account if there is no referrer.  <br/>- referrer_percent: the percentage (0 - 100) of the new user’s transaction fees not claimed by the blockchain that will be distributed to the referrer; the rest will be sent to the registrar. Will be multiplied by GRAPHENE_1_PERCENT when constructing the transaction.  <br/>- broadcast: true to broadcast the transaction on the network  | Registers a third party’s account on the blockckain. This function is used to register an account for which you do not own the private keys. When acting as a registrar, an end user will generate their own private keys and send you the public keys. The registrar will use this function to register the account on behalf of the end user. |    
| upgrade_account | - name: the name or id of the account to upgrade  <br/> - broadcast: true to broadcast the transaction on the network  | Upgrades an account to prime status. This makes the account holder a ‘lifetime member’. |
|create_account_with_brain_key  |  - from: the name or id of the account sending the funds <br/> - to: the name or id of the account receiving the funds <br/> - amount: the amount to send (in nominal units to send half of a BTS, specify 0.5) <br/> - asset_symbol: the symbol or id of the asset to send <br/> - memo: a memo to attach to the transaction. The memo will be encrypted in the transaction and readable for the receiver. There is no length limit other than the limit imposed by maximum transaction size, but transaction increase with transaction size <br/> - broadcast: true to broadcast the transaction on the network |Creates a new account and registers it on the blockchain.  |
| transfer |  - from: the name or id of the account sending the funds <br/> - to: the name or id of the account receiving the funds <br/>  - amount: the amount to send (in nominal units to send half of a BTS, specify 0.5) <br/> - asset_symbol: the symbol or id of the asset to send<br/> - memo: a memo to attach to the transaction. The memo will be encrypted in the transaction and readable for the receiver. There is no length limit other than the limit imposed by maximum transaction size, but transaction increase with transaction size<br/> - broadcast: true to broadcast the transaction on the network | Transfer an amount from one account to another. |
| transfer2 |  | This method works just like transfer, except it always broadcasts and returns the transaction ID along with the signed transaction. |
| whitelist_account | - authorizing_account: the account who is doing the whitelisting<br/> - account_to_list: the account being whitelisted<br/> - new_listing_status: the new whitelisting status<br/> - broadcast: true to broadcast the transaction on the network  | Whitelist and blacklist accounts, primarily for transacting in whitelisted assets. <br/>Accounts can freely specify opinions about other accounts, in the form of either whitelisting or blacklisting them. This information is used in chain validation only to determine whether an account is authorized to transact in an asset type which enforces a whitelist, but third parties can use this information for other uses as well, as long as it does not conflict with the use of whitelisted assets. |
| get_vesting_balances |account_name: An account name, account ID, or vesting balance object ID.  | Get information about a vesting balance object. |
| withdraw_vesting |  - witness_name: The account name of the witness, also accepts account ID or vesting balance ID type.  <br/> - amount: The amount to withdraw.  - asset_symbol: The symbol of the asset to withdraw. <br/> - broadcast: true if you wish to broadcast the transaction |Withdraw a vesting balance.  |
| get_account | account_name_or_id: the name or id of the account to provide information about |Returns information about the given account.  |
| get_account_id |account_name_or_id: the name of the account to look up  | Lookup the id of a named account. |
| get_account_history |  - name: the name or id of the account  - limit: the number of entries to return (starting from the most recent | Returns the most recent operations on the named account. This returns a list of operation history objects, which describe activity on the account. |
| approve_proposal |  - fee_paying_account: The account paying the fee for the op. <br/> - proposal_id: The proposal to modify. - delta: Members contain approvals to create or remove. In JSON you can leave empty members undefined. <br/> - broadcast: true if you wish to broadcast the transaction |Approve or disapprove a proposal.  |
|  |  |  |
